## 🟢 NEX-1001: Redesign Landing Page UI (Apple Developer Docs Style)

Status: Review
Priority: High
Source: `obsidian/kanban/specs/new-landing-page.md`

### Summary

Redesign 5 pages (Home, About, Counter, Portfolio, Dashboard) to use a clean, elegant UI inspired by Apple Developer Documentation. The app is a boilerplate showcase — the new UI should make the stack look professional and polished, not replace its demo nature.

### Context

- Current pages are boilerplate placeholder content with minimal styling
- Stack: Next.js App Router, Tailwind CSS v4, Clerk auth, next-intl (en/fr)
- No `prd.md` exists — this is a developer-facing boilerplate showcase project
- Reference `@clark/nextjs` mentioned in spec is not installed in `package.json` — treat as inspiration only

### Design Decisions (from grilling session)

- **App type**: Boilerplate showcase
- **Theme**: System light/dark (`prefers-color-scheme`) via Tailwind v4 dark mode
- **Layout**: Persistent left sidebar nav (Apple Developer Docs style) — replaces current `BaseTemplate` structure
- **Style**: Minimal, clean, neutral gray scale

### Scope

**1. Home** (`src/app/[locale]/(marketing)/page.tsx`)
- Add hero section: headline + subtext + CTA button (top)
- Below hero: feature grid showing boilerplate stack
- Replace current plain feature list + Sponsors

**2. About** (`src/app/[locale]/(marketing)/about/page.tsx`)
- Redesign layout to match Apple developer docs style
- Keep existing i18n content (`About.about_paragraph`)

**3. Counter** (`src/app/[locale]/(marketing)/counter/page.tsx`)
- Keep `CounterForm` + `CurrentCount` functionality unchanged
- Redesign layout and styling only

**4. Portfolio** (`src/app/[locale]/(marketing)/portfolio/page.tsx`)
- Redesign to card grid layout
- Keep mock data (6 hardcoded items) — no DB connection needed

**5. Dashboard** (`src/app/[locale]/(auth)/dashboard/page.tsx`)
- Replace `<Hello />` with stats overview: counter value, portfolio count, user info (from Clerk)
- No welcome/hero section — start with stats cards directly

### Out of Scope

- Connecting portfolio to real DB
- Adding new pages
- Changing counter or auth logic
- `@clark/nextjs` package integration

### Open Questions

- Font: system font stack (`-apple-system, BlinkMacSystemFont`) vs. custom (e.g. Inter)?
- Sidebar mobile behavior: hamburger menu or bottom nav?
- Accent color: neutral gray-only or include an accent color?
- Sidebar: create new shared layout component or modify existing `BaseTemplate`?

### Acceptance Criteria

- [ ] All 5 pages render without errors in both light and dark mode
- [ ] Layout uses persistent left sidebar matching Apple developer docs aesthetic
- [ ] Home has hero section + feature grid below
- [ ] Portfolio shows card grid (6 mock items)
- [ ] Dashboard shows stats cards (counter value, portfolio count, Clerk user info)
- [ ] All user-visible strings go through `getTranslations` / `useTranslations` — no hardcoded copy
- [ ] TypeScript: no `any`, no type errors (`bun run check:types` passes)
- [ ] Lint passes (`bun run lint`)
- [ ] Responsive: pages usable on mobile viewport

### Validation

- `bun run lint` — no errors
- `bun run check:i18n` — no missing keys
- Visual check: all 5 pages in light + dark mode
- Mobile viewport check on each page
