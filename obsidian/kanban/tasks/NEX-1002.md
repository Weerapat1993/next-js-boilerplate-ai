## 🟢 NEX-1002: Settings appearance — theme mode setting

Status: Done
Priority: Medium
Source: `obsidian/kanban/specs/settings-appearance-theme.md`

### Summary

New page `/[locale]/(auth)/dashboard/settings/appearance` lets logged-in user pick `light` / `dark` / `system` theme. Preference persisted per-user in Postgres (Drizzle), not cookie/localStorage. No flash-of-wrong-theme on load (SSR-aware).

### Context

- Dark mode currently OS-only via `@custom-variant dark (&:where(.dark, .dark *))` in `src/styles/global.css:5` — no UI, no persistence, no override.
- New group under existing `dashboard` auth group.

### Domain model

New table `userPreferences` in `src/models/Schema.ts`:

```ts
export const userPreferences = pgTable('user_preferences', {
  id: uuid('id').primaryKey().defaultRandom(),
  clerkUserId: text('clerk_user_id').notNull().unique(),
  themeMode: text('theme_mode', { enum: ['light', 'dark', 'system'] })
    .notNull()
    .default('system'),
  createdAt: timestamp('created_at', { mode: 'date' }).defaultNow().notNull(),
  updatedAt: timestamp('updated_at', { mode: 'date' })
    .defaultNow()
    .$onUpdate(() => new Date())
    .notNull(),
});
```

One row per user (`clerkUserId` unique), not generic key-value prefs blob.

### Scope

- Schema: add `userPreferences` table, generate + review migration.
- Read path: `[locale]/(auth)/dashboard/layout.tsx` server-side `auth()` → query pref (default `'system'`) → resolve `light`/`dark` class on `<html>` server-side; for `system`, no forced class + tiny blocking inline `<script>` reading `matchMedia('(prefers-color-scheme: dark)')` before paint.
- Write path: server action `updateThemeMode` in `src/app/[locale]/(auth)/dashboard/settings/appearance/actions.ts` — `auth()` for `clerkUserId`, Zod-validated input, upsert into `userPreferences`. On success, `revalidatePath` / navigation so layout re-reads server-side (no client theme context).
- Zod schema for `themeMode` in `src/validations/`.
- Page `src/app/[locale]/(auth)/dashboard/settings/appearance/page.tsx`: `PageProps` with `params: Promise<{ locale: string }>`, `setRequestLocale`; radio/segmented control Light/Dark/System pre-selected from server-fetched pref; submits via server action.
- Meta (title/description) defined once in `dashboard/layout.tsx`, not per-page.
- i18n namespace `SettingsAppearancePage` in `src/locales/en.json` and `fr.json`: `page_title`, `theme_light`, `theme_dark`, `theme_system`, `save_button`, `save_success`.

### Out of Scope

- Per-component theming beyond existing Tailwind `dark:` classes.
- Anonymous/public theme preference — public pages stay OS-only.
- Additional appearance settings (font size, accent color, etc.).

Fixed by having `updateThemeMode` set a `theme_mode` cookie on save, and `getThemeMode()` check that cookie first (works on every route, no middleware dependency) before falling back to the `auth()` + DB lookup path. Fixed by adding `src/components/SignOutThemeResetButton.tsx` (clears the cookie via the Cookie Store API on the sign-out click, wired into `dashboard/layout.tsx` in place of the raw `SignOutButton`) with a new browser test `SignOutThemeResetButton.test.tsx`.
